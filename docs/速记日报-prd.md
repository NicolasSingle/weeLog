## Executive Summary

速记日报是一款跨Windows/Mac平台的极致轻量桌面工作管理工具，核心定位为"零干扰的个人工作记录与AI汇总工具"。产品通过全局悬浮小窗口实现10秒快速写日报、一键管理待办事项，结合AI自动生成周报月报功能，全面解放职场人士的工作汇报时间。目标用户为需要每日写日报、每周写周报的职场人士和开发者，核心原则是轻量、快速、隐私优先、不打扰、一站式工作管理。

产品采用双窗口架构：悬浮小窗口用于快速录入（全局快捷键唤起，始终置顶），主窗口用于完整管理（待办、日报、周报月报、AI汇总、设置）。所有数据采用本地SQLite存储，确保用户隐私安全。AI功能集成Minimax大模型，支持流式输出和自定义提示词。

## Problem Statement

**Current Situation**: 
- 职场人士每天花费10-30分钟撰写日报，周报月报更是耗时
- 工作中临时想法和待办事项分散在微信、邮件、笔记等多个平台
- 写日报时需要回忆全天工作内容，导致日报质量参差不齐或遗漏
- 现有工具普遍臃肿，安装包大、占用资源多、界面复杂

**Proposed Solution**:
速记日报通过"悬浮窗口 + 快捷键"实现随时随地零干扰记录，待办事项与日报自动关联，AI基于原始记录智能生成周报月报。

**Business Impact**:
- 预计为每位用户每天节省15-30分钟汇报时间
- 提升日报完整性和真实性，减少回忆遗漏
- 极低资源占用（目标<80MB内存），不影响其他工作

- 高度：40px

- 待办数量角标：红色圆形，位于「待办」标签右侧

- 文本输入框：距顶部8px，左右各12px，高度120px最小

**待办标签页：**
- 待办输入框：距顶部8px，高度36px，placeholder"添加待办..."
- 添加快捷键：Enter添加，光标清空
- 待办列表：输入框下方，可滚动区域
- 待办项高度：40px，包含复选框、内容、剩余时间
- 拖拽手柄：待办项左侧，6点图标
- 右键菜单：编辑、删除、设置截止时间

**窗口状态记忆：**
- 记住每个标签页的最后滚动位置
- 记住每个标签页的激活状态
- 记住窗口大小和位置

**Feature 2: 待办事项管理**

**待办状态机：**
```
[新建] → [未完成] → [已完成]
              ↓
         [已过期]（超过截止时间且未完成）
```

**待办属性：**
| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | UUID | 是 | 唯一标识 |
| title | string | 是 | 标题，1-200字符 |
| description | string | 否 | 描述，0-2000字符 |
| dueDate | datetime | 否 | 最晚完成时间 |
| priority | enum | 否 | high/medium/low，默认medium |
| tags | string[] | 否 | 标签数组 |
| status | enum | 是 | pending/completed/expired |
| createdAt | datetime | 是 | 创建时间 |
| completedAt | datetime | 否 | 完成时间 |

**优先级颜色：**
- 高（high）：#F53F3F，红色，左侧4px边框
- 中（medium）：#FF7D00，橙色，左侧4px边框  
- 低（low）：#00B42A，绿色，左侧4px边框

**分类视图：**
- 今日待办：dueDate为今天或无截止时间且status=pending
- 本周待办：dueDate在本周一至周日之间
- 全部待办：无过滤，显示所有
- 已完成：status=completed，按完成时间倒序

**自动同步日报规则：**
- 当待办状态从pending变为completed时
- 自动在当天日报末尾添加一行：`✓ 完成了：[待办标题]`
- 如果当天无日报，则自动创建一条

**提醒机制：**
- 到期前1小时发送桌面通知
- 通知内容："待办即将到期：[标题]"
- 点击通知：唤起主窗口并定位到该待办

**Feature 4: AI汇总功能**

**模型配置：**
| 参数 | 说明 | 默认值 |
|------|------|--------|
| apiKey | Minimax API密钥 | 用户配置 |

**提示词模板（默认）：**
```
你是一位专业的工作汇报助手。请根据以下日报和待办记录，生成一份简洁专业的周报/月报。

要求：
1. 结构清晰：先总结概览，再按项目/分类详细说明

3. 语言简洁：每段不超过3句话
4. 格式规范：使用 Markdown 标题和列表

---
日报记录：
{todayReports}

已完成待办：
{completedTodos}

请生成{reportType}：
```

1. 用户选择时间范围、报告类型（周报/月报）
2. 点击"生成"按钮
3. 显示进度条和流式输出区域
4. AI逐字输出结果，实时渲染
5. 用户可随时点击"停止"中断
6. 生成完成后可编辑、复制、导出

**导出格式：**
- Markdown（.md）
- Word（.docx）
- 纯文本（.txt）

## Technical Constraints

### Performance

| 指标 | 要求 |
|------|------|
| 冷启动时间 | ≤3秒 |
| 热启动时间 | ≤1秒 |
| 内存占用（空闲） | ≤100MB |
| 悬浮窗口响应 | ≤100ms |
| 日报搜索响应 | ≤500ms |
| AI生成速度 | 取决于网络和模型 |

### Security

- **数据本地存储**：所有数据存储在用户本地，不上传服务器
- **API密钥加密**：存储时使用系统密钥链加密
- **隐私优先**：不收集任何用户行为数据
- **无网络要求**：离线时日报和待办功能完全可用（AI汇总需联网）

### Integration

- **Minimax API**：用于AI汇总功能，需用户配置API密钥
- **系统通知**：使用系统原生通知接口
- **系统托盘**：使用Electron/Tauri原生API
- **快捷键**：使用全局快捷键注册（跨平台库）

## Dependencies & Blockers

**Dependencies:**
- **Minimax API**：AI汇总功能必需，需提前申请API密钥
- **系统托盘API**：各平台差异较大，需针对性测试
- **全局快捷键注册**：需要管理员权限，需处理UAC提示

**Known Blockers:**
- **Windows快捷键注册**：Ctrl+Shift+D可能被其他软件占用 → 提供备选快捷键
- **macOS沙箱权限**：如果从App Store分发需要额外权限申请 → 使用公证分发

### Core User Journey Map

```
用户旅程：快速记录日报

Step 1: 工作中想到要点
├── 触发：正在coding或处理工作
└── 动作：无需切换应用

Step 2: 按快捷键唤起浮窗
├── 按下：Ctrl+Shift+D / Cmd+Shift+D
├── 响应：<100ms弹出浮窗（屏幕右上角）
└── 状态：输入框自动聚焦，光标闪烁

Step 3: 快速输入日报
├── 输入：多行文本，支持#标签
├── 语法：Enter提交，Shift+Enter换行
└── 反馈：输入流畅，无卡顿

Step 4: 提交并继续工作
├── 按下：Enter
├── 响应：显示"已保存✓"（1秒）
└── 结果：浮窗自动隐藏，无打扰

Step 5: 日后生成周报
├── 打开：主窗口 → 周报页面
├── 选择：时间范围（本周）
├── 点击：生成按钮
└── 结果：AI流式输出周报，可编辑/导出
```
